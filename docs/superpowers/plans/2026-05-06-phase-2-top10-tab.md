**Step 2 — update `ngOnChanges`:**

```ts
ngOnChanges(): void {
  this.topRows = this.data.slice(0, 10);
  const hasSecondary = !!(this.secondaryData && this.secondaryData.length > 0);
  const secondaryRows = hasSecondary ? (this.secondaryData as BarDatum[]).slice(0, this.topRows.length) : [];

  const primary = resolvePrimary();

  const series: any[] = [{
    name: this.primaryLabel,
    type: 'bar',
    stack: 'rank',
    data: this.topRows.map(d => d.value),
    barMaxWidth: 24,
    barCategoryGap: '20%',
    itemStyle: { color: primary },
    emphasis: { focus: 'series' },
  }];

  if (hasSecondary) {
    series.push({
      name: this.secondaryLabel,
      type: 'bar',
      stack: 'rank',
      data: secondaryRows.map(d => d.value),
      barMaxWidth: 24,
      itemStyle: { color: primary, opacity: 0.30 },
      emphasis: { focus: 'series' },
    });
  }

  this.options = {
    tooltip: { trigger: 'axis', axisPointer: { type: 'shadow' } },
    legend:  hasSecondary ? { data: [this.primaryLabel, this.secondaryLabel], right: 0, top: 0 } : undefined,
    grid:    { left: 100, right: 30, top: hasSecondary ? 40 : 20, bottom: 30 },
    xAxis:   { type: 'value' },
    yAxis:   { type: 'category', data: this.topRows.map(d => d.label), inverse: true, axisLabel: { fontSize: 11 } },
    series,
  };
}
```

The single-series existing usage (Overview's Top Airlines) is unchanged — when `secondaryData` is undefined, the `if` branch is skipped and the chart renders as before.

echarts surfaces the y-axis category on click regardless of which stacked series received the click, so the existing `event.name` extraction continues to work.

## Common pitfalls (Phase 2 specific)

1. **Forgetting `queryParamsHandling: 'preserve'` on the tab nav.** Without it, clicking a tab strips `?airport=...` and the destination tab kicks back to `applyDefault()`. Filter state must follow the user across tabs.
2. **Unbalanced primary/secondary lengths in `topFlightsServiced$` vs `topFlightsGap$`.** echarts will render whatever it gets — including misaligned y-axis categories. Always derive both from the same `r.flights.items` array.
3. **`secondary - primary` gap going negative.** If a flight is over-serviced (data anomaly), `requested - serviced < 0` and echarts renders below the axis. The `Math.max(0, …)` guard in the orchestrator is load-bearing.
4. **Forgetting to add `TableModule` to SharedModule.** `<p-table>` will silently render as `<p-table>` with no styles applied. Confirm by inspecting the rendered HTML — should see `.ui-table` class on the container.
5. **`responsiveLayout="scroll"` vs `"stack"`.** Scroll keeps columns aligned but introduces horizontal scrollbars on narrow widths; stack collapses to one row per cell on mobile. Use `"scroll"` for an analytics dashboard where column alignment matters more than mobile ergonomics.
